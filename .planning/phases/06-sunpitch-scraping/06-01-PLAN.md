**sunpitch.ts** — Export `scrapeSunPitch(browser: Browser, url: string): Promise<ScrapeResult>`. Implement the dual-strategy approach:

1. Set up `page.on('response')` BEFORE goto to capture any JSON API responses containing proposal data. Store them in `capturedApiPayloads: Array<{ url: string; data: unknown }>`. Heuristic for what qualifies: response Content-Type includes 'application/json', response status is 2xx, and the JSON body contains at least one of: `systemSize`, `monthlyProduction`, `production`, `proposal`, `customer`, `clientName`.

2. `await page.goto(url, { waitUntil: 'domcontentloaded', timeout: 20000 })`

3. After goto, check `page.url()` — if it no longer contains '/facing/proposals/', immediately return `{ status: 'error', data: null, missingFields: [], message: 'SunPitch redirected to login or an unexpected page. The proposal URL may require authentication.' }`

4. Wait for content: `await page.waitForSelector('[data-testid], [data-proposal], main, .proposal, article', { timeout: 15000 }).catch(() => null)` — soft wait, continue even if nothing found.

5. If `capturedApiPayloads` has data, call `extractFromApiResponse(capturedApiPayloads)` to map to ProposalFormValues. Otherwise call `extractFromDOM(page)`.

6. Both `extractFromApiResponse` and `extractFromDOM` must:
   - Return `ScrapeResult`
   - Log what they found (add `console.log('[scraper]', ...)` lines — helps debugging before DOM is known)
   - Call `normalizeMonthlyArray()` on any monthly data
   - Populate `missingFields` with dot-notation paths of fields NOT found (e.g. `'customer.name'`, `'system.monthlyProductionKwh'`)
   - Check hard-error threshold: if `customer.name` AND `system.systemSizeKw` AND `system.monthlyProductionKwh` are ALL missing from data, return `status: 'error'` with message "Could not extract minimum required data (customer name, system size, and monthly production). The SunPitch proposal layout may have changed."
   - Otherwise return `status: 'success'` (all fields found) or `status: 'partial'` (some missing)

7. `normalizeMonthlyArray(raw: (string | number | null | undefined)[]): string[]` — pad/trim to exactly 12 entries using `'0'` as fill.

**Important:** Since SunPitch's actual DOM structure is unknown until a real URL is tested, `extractFromDOM` should use flexible, resilient selectors. Try these in order for each field:

- Monthly production: look for a chart or table with 12 numeric values; try `page.locator('[class*="month"], [class*="production"], td').allTextContents()`
- For each extraction attempt, wrap in try/catch and push path to `missingFields` on failure

```typescript
import { NextRequest } from 'next/server';
import { chromium } from 'playwright';
import { scrapeSunPitch } from '@/lib/scraper/sunpitch';
import type { ScrapeResult } from '@/lib/scraper/types';

// REQUIRED: Playwright cannot run in Edge runtime
export const runtime = 'nodejs';

export async function POST(request: NextRequest): Promise<Response> {
  let body: { url?: string };
  try {
    body = await request.json();
  } catch {
    return Response.json({ status: 'error', message: 'Invalid JSON body', data: null, missingFields: [] }, { status: 400 });
  }

  const { url } = body;
  if (!url || typeof url !== 'string') {
    return Response.json({ status: 'error', message: 'Missing url', data: null, missingFields: [] }, { status: 400 });
  }

  const browser = await chromium.launch({
    headless: true,
    args: ['--no-sandbox', '--disable-setuid-sandbox', '--disable-dev-shm-usage'],
  });

  try {
    const result: ScrapeResult = await scrapeSunPitch(browser, url);
    return Response.json(result);
  } catch (err) {
    const message = err instanceof Error ? err.message : 'Unknown scrape error';
    const errorResult: ScrapeResult = { status: 'error', data: null, missingFields: [], message };
    return Response.json(errorResult, { status: 200 }); // 200 — client reads status field, not HTTP code
  } finally {
    await browser.close();
  }
}
```

Note: Return HTTP 200 even on scrape failures (not 500) — the client reads `result.status`, not the HTTP status. Only structural errors (bad JSON body, missing url param) return 4xx.
