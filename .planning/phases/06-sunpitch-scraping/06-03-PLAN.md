  <how-to-verify>
**Before verifying — update selectors in sunpitch.ts:**

If the scrape returns partial/error with many missing fields, inspect the real SunPitch DOM:
1. Open the SunPitch URL in Chrome DevTools
2. Check the Network tab for JSON API calls during page load — look for responses containing system or customer data
3. Update `extractFromApiResponse()` in `src/lib/scraper/sunpitch.ts` with the real API endpoint pattern
4. If no clean JSON API, update `extractFromDOM()` with selectors from the real DOM

**Then verify the full flow:**

1. Start dev server: `npm run dev` (port 3001)
2. Navigate to http://localhost:3001/enter
3. Verify: Two tabs visible at the top — "Paste SunPitch URL" (default active) and "Manual Entry"
4. Paste the SunPitch URL from Task 1 into the URL field
5. Click "Import Data"
6. Verify: Cycling status messages appear ("Opening SunPitch proposal...", "Extracting customer information...", etc.)
7. Verify: After 5-15 seconds, the page auto-switches to the Manual Entry tab
8. Verify: Fields that were successfully scraped have a light green tint/ring
9. Verify: Fields the scraper missed have an amber/yellow tint/ring
10. Verify: Click on a green field and edit it — the green highlight should clear
11. Verify: Refresh the page — scraped values should still be there (draft persisted)
12. Verify: Fill in any amber (missing) fields manually, then click "Generate Proposal" — confirm the proposal renders correctly at /proposal

**If scraping fails completely:**
- Check browser console / server logs for `[scraper]` log messages
- Note which strategy ran (API interception vs DOM) and what was found
- Update sunpitch.ts selectors based on what the real DOM actually contains
- Re-test after selector updates

**Error scenario test:**
- Try typing a bad URL (e.g., "https://example.com") — should see inline error instantly, no network request
- Try a correctly-formatted-but-wrong URL — should see error from scraper within 20 seconds
