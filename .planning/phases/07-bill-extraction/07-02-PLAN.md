<task type="auto">
  <name>Task 2: Add /proposal → /enter redirect</name>
  <files>src/app/proposal/page.tsx</files>
  <action>
    IMPORTANT: Check if src/app/proposal/page.tsx already exists before writing.

    Run: `ls src/app/proposal/` to see what's there.

    The proposal renderer lives at src/app/proposal/ — it IS the page.tsx in that directory. We cannot overwrite it with a redirect.

    The correct approach: add a Next.js redirect in next.config.ts (or next.config.js) using the `redirects` async function.

    Read the current next.config.ts/next.config.js file first, then add to it:

    ```typescript
    async redirects() {
      return [
        {
          source: '/proposal',
          destination: '/enter',
          permanent: true,  // 301
        },
      ];
    },
    ```

    CRITICAL: The proposal renderer uses /proposal?d=<base64> URL pattern. The redirect must NOT intercept URLs with the `d` query param. Check if Next.js redirects match on path only (not query) — they do by default, so `/proposal?d=...` will also be redirected. This is WRONG.

    Solution: Use a has condition to only redirect when there is NO `d` param:
    ```typescript
    {
      source: '/proposal',
      destination: '/enter',
      permanent: false,  // 307 — safer while testing
      missing: [{ type: 'query', key: 'd' }],  // only redirect when d param is absent
    },
    ```

    This ensures:
    - /proposal (no params) → redirects to /enter
    - /proposal?d=<base64> → serves the proposal renderer as normal

    Files to modify: next.config.ts (or next.config.js — read which exists first).
    Do NOT modify src/app/proposal/page.tsx.
  </action>

  </verify>
