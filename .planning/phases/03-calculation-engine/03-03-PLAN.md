Create `src/app/actions/calculate.ts` following the exact pattern of `get-defaults.ts`.

The server action accepts `ProposalFormValues` (all-string form data), converts to `ProposalInputs` (Decimal), calls `calculateProposal(inputs, config)`, then serializes all Decimal fields to strings for JSON transport across the server/client boundary.

Key decisions (per CONTEXT.md locked decisions):
- Server action is async — fetches `getAlbertaConfig()` before calling pure `calculateProposal()`
- All form string values converted via `d(value)` — never through `Number(value)` or `parseFloat()`
- Decimal → string serialization via `JSON.stringify(outputs, decimalReplacer)` then `JSON.parse()` — return a plain object with strings

Define a `SerializedProposalOutputs` type where all Decimal fields become `string` and Decimal[] become `string[]`. MonthlyNetMetering serializes each field to string. CarbonCreditProjection serializes similarly.

Server action signature:
```typescript
'use server';

export type SerializedMonthlyNetMetering = {
  month: string; consumptionKwh: string; productionKwh: string;
  gridBuyKwh: string; surplusSoldKwh: string; costToBuy: string; revenueEarned: string;
};

export type SerializedSavingsSummary = {
  utilityAvoided: string; netMeteringRevenue: string; carbonCredits: string;
  cashBack: string; totalSavings: string; netSavingsAfterCost: string;
};

export type SerializedProposalOutputs = {
  solarOffsetPercent: string;
  twentyYearUtilityCost: string;
  twentyYearSavings: string;
  monthlyNetMetering: SerializedMonthlyNetMetering[];
  annualGridPurchaseCost: string;
  annualSellRevenue: string;
  carbonCredits: {
    annualCo2Avoided: string; tenYearPayoutLow: string; tenYearPayoutHigh: string;
    benchmarkSchedule: Array<{ year: number; pricePerTonne: string; payoutLow: string; payoutHigh: string; }>;
  };
  cashPurchase: { twentyYear: SerializedSavingsSummary; thirtyYear: SerializedSavingsSummary; };
  financeOption: { monthlyAllInCost: string; totalMonthlyPayment: string; };
  utilityProjection20Year: string[];
  utilityProjection30Year: string[];
};

export async function runCalculation(formValues: ProposalFormValues): Promise<SerializedProposalOutputs>
```

Implementation approach:
1. Fetch `const config = await getAlbertaConfig()`
2. Build `ProposalInputs` by converting each string field with `d(value)`, `parseInt(value, 10)` for financeTermMonths
3. Call `const outputs = calculateProposal(inputs, config)`
4. Serialize by mapping each Decimal field to `.toString()` — do this explicitly (not via JSON.stringify replacer, which loses structure). Write a manual mapping so the return type is fully typed.

Import `ProposalFormValues` from `@/lib/form/schema`. Import `ProposalInputs, MonthlyValues` from `@/lib/types`. Import `calculateProposal` from `@/lib/engine/calculate`. Import `getAlbertaConfig` from `@/lib/config/alberta`. Import `d` from `@/lib/decimal`.
